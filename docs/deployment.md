# 部署指南

這份文件給「要把工具發給社工使用」的機構負責人或 IT 人員。

整個部署是**一次性**的，做完之後社工只要重複「產生證書」這個動作即可。

## 第一階段：準備檔案（5 分鐘）

### 1.1 取得 HTML 檔

從本 repo 的 [dist/cert-generator.html](../dist/cert-generator.html) 直接下載，或從 [Releases](../../releases) 抓最新版本。

這是一個 ~1.3 MB 的單一 HTML 檔，所有相依函式庫都已內嵌。

### 1.2 準備印章圖

需要兩張：

- **機構用印**：機構名稱左側那顆方章。建議 PNG 去背、約 600×600 像素
- **主任簽名章**：主任的簽名章。建議 PNG 去背、長條型

可以掃描實體印章再去背，或請印章供應商提供電子版。

### 1.3 確認字型

工具預設使用「標楷體」。Windows 從 XP 到 11 都內建 DFKai-SB（標楷體），不用額外安裝。macOS 內建 BiauKai 作為替代。

如果機構電腦用的是特殊版本的 Windows（例如 LTSC 精簡版）可能要確認，但這幾乎不會發生。

## 第二階段：建立發放紀錄試算表（10 分鐘）

> 這一步是「選用」的——不做也能正常產 PDF 與登錄表，只是不會自動記錄發放歷史。

### 2.1 建立試算表

到 [Google 試算表](https://sheets.google.com) 建一張空白試算表，命名為「研習證明發放紀錄」（名稱隨意）。

### 2.2 設定 Apps Script

1. 試算表上方選單「**擴充功能 → Apps Script**」
2. 把編輯器裡原本的 `function myFunction() {}` 全部刪掉
3. 開啟 [docs/apps-script.gs](apps-script.gs)，把全部內容貼進去
4. 按 💾 圖示或 Ctrl+S 儲存

### 2.3 部署為 Web App

1. 右上角「**部署 → 新增部署作業**」
2. 左側齒輪選「**網頁應用程式**」
3. 「執行身分」選**你自己**
4. 「存取權」選「**所有人**」（不需要登入 Google 即可送資料進來）
5. 按「部署」

第一次部署會跳出「授權」對話框，按指示同意。中間可能會看到「Google 未驗證此應用程式」的警告，按「**進階 → 前往未驗證的專案**」即可（因為這是你自己寫的腳本）。

部署完成後會給你一個網址，格式類似：

```
https://script.google.com/macros/s/AKfycb.../exec
```

把這個網址記下來。

## 第三階段：發給社工使用

### 3.1 第一次設定（一次性，由負責人或社工自己做）

1. **下載 HTML 檔**到電腦桌面（不要直接從雲端開啟，下載到本機最穩）
2. **雙擊用瀏覽器打開**（建議 Chrome 或 Edge）
3. 點到「**設定**」分頁
4. **上傳兩張印章**：分別上傳機構用印和主任簽名章。印章會存在瀏覽器本機
5. **貼上 Apps Script 網址**（步驟 2.3 取得的那個）
6. **貼上試算表的瀏覽網址**（瀏覽器網址列上 `https://docs.google.com/spreadsheets/d/...` 那段）
7. 按「**測試寫入**」確認試算表能成功收到資料
8. **匯入員工主檔**：到「員工主檔」分頁，把員工資料 Excel 匯入

設定完成。

### 3.2 例行使用

之後每次有訓練，社工只要：

1. 開啟 HTML 工具
2. 到「產生證書」分頁
3. 勾選參加員工
4. 填訓練資訊與課程
5. 按「全部產生」

完整操作見 [user-guide.md](user-guide.md)。

## 部署選項：放在哪裡？

**選項 A：放本機桌面**
- 最單純，每個社工的電腦各一份
- 員工主檔資料分散，但既然單機操作就無妨
- 適合：1 個人負責，不換電腦

**選項 B：放機構共用雲端硬碟**
- 從雲端下載到本機後再開啟（不要直接從預覽開）
- 升級時統一替換雲端版本即可
- 適合：可能換人接手或多台電腦

**選項 C：自架網頁伺服器**
- 對社工來說沒差，但對你來說 OK
- 注意：原本 HTML 跑 `file://` 與 `https://` 的行為略有差異，建議測試過再部署

## 升級流程

未來需要更新工具時：

1. 從 [Releases](../../releases) 下載新版 HTML
2. 替換社工電腦上的舊版本
3. 員工主檔、印章、設定值都會保留（存在瀏覽器 localStorage，跟 HTML 檔分離）

## 疑難排解

**問題：開啟 HTML 後一片空白**
- 檢查瀏覽器版本，太舊的 IE / Edge Legacy 不支援
- F12 開 Console 看錯誤訊息

**問題：「設定」頁的字型偵測顯示紅色「找不到楷書字型」**
- 確認系統地區設定是「中文（繁體，台灣）」
- 控制台 → 地區 → 變更系統地區設定

**問題：產生 PDF 中文變方框**
- 同上，標楷體未安裝
- 或字型 fallback 失敗，產出有亂碼，先確認系統字型

**問題：寫入 Google 試算表失敗**
- 到「設定」分頁按「測試寫入」確認 Apps Script 是否運作
- 確認 Web App 部署的「存取權」設為「所有人」
- 確認部署網址（exec 結尾）沒貼錯

**問題：印章圖太大/太小**
- 在「設定」重新上傳，建議用 600×600 像素的 PNG
- 如果印章圖檔本身就奇怪（例如背景不透明），先用繪圖軟體處理
